# 매쓰홀릭 Android 접근성·연동 기술 문의 초안

제목: 학생용 Android 앱의 키오스크 연동을 위한 접근성 및 공식 로그인·로그아웃 방식 문의

안녕하세요. 학원 내 전용 태블릿에서 학생 개인계정의 로그인·채점·로그아웃을 안전하게 운영하기 위한 키오스크 보조 앱을 검토하고 있습니다. 자동화 사용 가능 여부는 별도 문의를 통해 허용된다고 안내받았습니다.

다음 환경에서 Android 접근성 API를 이용한 실기 검증을 진행했습니다.

- 매쓰홀릭 package: `com.matholic.mathapp`
- 앱 versionName/versionCode: 7.5.1 / 770961
- 기기 A: Samsung SM-P610, Android 13, One UI 5.1.1
- 기기 B: Samsung SM-P613, Android 14, One UI 6.1

두 기기에서 아이디·비밀번호 입력란과 로그인·로그아웃 버튼은 일부 의미 노드로 노출됐으나 다음 필수 요소는 접근성 상태·Action을 제공하지 않았습니다.

1. `로그인 정보 기억하기`는 시각적 체크 전·후 모두 `checkable=false`, checked `false`로 보고되어 실제 상태를 판독할 수 없습니다.
2. 홈 화면의 메뉴와 학생 이름 진입점에 click Action, 접근성 이름 및 안정적인 resource ID가 없습니다.
3. 문제풀이 화면의 좌측 상단 `<` 복귀 아이콘이 접근성 트리에 노출되지 않습니다.
4. 홈 우측 상단 QR·학습번호·채팅 아이콘에 접근성 label과 click Action이 없습니다.
5. 학생 이름을 눌러 여는 프로필 팝업과 팝업 안 `로그아웃` 버튼은 노출되지만, 프로필 팝업 진입점 자체는 Action을 제공하지 않습니다.

좌표·이미지 인식은 화면 변경 시 다른 학생 계정을 잘못 처리할 위험이 있어 사용하지 않으려 합니다. 아래 중 공식적으로 지원 가능한 방법이 있는지 확인 부탁드립니다.

1. 위 요소들에 Android/WebView 접근성 role, accessible name과 click Action 제공
2. 기억하기 상태에 checkbox role과 `aria-checked` 또는 동등한 상태 제공
3. 각 필수 요소에 버전 간 안정적인 resource ID 또는 automation ID 제공
4. 학생 로그인용 공식 SSO, 일회성 토큰, deep link 또는 인증 API 제공
5. 서버 세션까지 종료되는 공식 로그아웃 API 또는 deep link 제공
6. 로그인 후 학생 표시명과 고유 ID를 안전하게 확인할 공식 방식 제공
7. 수정 가능 여부, 예상 버전과 사전 시험 채널 제공 가능 여부

자격정보, 학생 실명, 답안과 성적은 외부로 전송하거나 로그에 저장하지 않는 실패폐쇄 방식으로 설계할 예정입니다. 필요한 기술자료나 권장 연동 방식이 있다면 안내 부탁드립니다.
