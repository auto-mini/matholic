# RC02 잔여 운영 인수시험 체크리스트

작성일: 2026-07-24 (Asia/Seoul)

이 문서는 USB 디버깅을 끈 A 태블릿에서 사용자가 혼자 남은 두 시험을 수행하기 위한 절차다. 계정값, 관리자 PIN과 QR 원문은 기록하거나 전송하지 않는다.

## 현재 시작 상태

- A 화면: `QR_READY`
- 설치본: Kiosk `0.5.0-rc02`, Web POC `0.3.5-rc02`
- 휴대폰: 현재 유효한 신규 QR 사진 보유
- USB 디버깅: 꺼짐
- 전용기기 잠금: 활성

현재 QR은 **프린터 시험 직전까지 재발급하지 않는다**. 재발급하면 휴대폰의 기존 QR이 즉시 무효가 된다.

## A. 1~2시간 연속 운전

최소 60분, 권장 120분이다. PC가 아닌 일반 충전기에 연결하고 안정적인 Wi-Fi를 사용한다.

### 시작

- [ ] 화면 우측 상단이 `QR_READY`
- [ ] 카메라 미리보기가 멈추지 않음
- [ ] 오류나 PIN 화면이 없음
- [ ] 시작 시각 기록: `____시 ____분`

### 종료 왕복

1. 휴대폰에 저장한 현재 QR을 A의 전면 카메라에 보여준다.
2. 자동 로그인된 웹 화면에서 좌상단 `학습지`를 누른다.
3. 중앙의 `학습하기`를 눌러 문제 화면에 들어간다.
4. Android 뒤로가기를 누르지 않는다.
5. 우측 하단 `채점 끝내기`를 누른다.
6. 약 10초 안에 QR 카메라 화면으로 돌아오는지 확인한다.

합격 기준:

- [ ] Web 화면 자동 진입
- [ ] 문제 화면 진입
- [ ] `채점 끝내기` 후 자동 로그아웃
- [ ] 최종 `QR_READY`
- [ ] 잠금·PIN·오류 화면 없음

## B. 실제 프린터 시험

신뢰할 수 있는 로컬 프린터가 있을 때만 수행한다. 프린터가 없으면 `SKIP — 실제 프린터 없음`으로 기록하며, 프린터 잔류 위험은 미검증으로 남는다. 공유 프린터, 클라우드 프린터와 PDF 저장은 사용하지 않는다.

### QR 교체와 출력

1. QR 대기 화면의 `관리자`를 누르고 관리자 PIN을 입력한다.
2. 등록 학생 `테스트`를 선택한다.
3. `QR 폐기·재발급`을 한 번만 누른다.
4. 새 QR이 나타나면 즉시 휴대폰으로 촬영한다.
5. 휴대폰의 이전 QR 사진은 삭제하거나 `폐기`로 표시해 혼동하지 않는다.
6. `현재 표시 QR 인쇄`를 누른다.
7. 경고를 확인하고 신뢰하는 로컬 프린터 한 대만 선택한다.
8. 한 장만 출력하고 인쇄 대기열에 작업이 남지 않았는지 확인한다.

인쇄가 실패해도 4단계에서 촬영한 새 QR은 유효하다. QR 사진을 잃어버린 경우에만 다시 재발급한다.

### 종이 QR 재인식

1. Kiosk로 돌아온다. PIN 화면이 나오면 관리자 PIN을 입력한다.
2. 활성 수업이 유지 중이면 `QR 대기 화면으로 복귀`를 누른다.
3. 출력한 종이 QR을 전면 카메라에 보여준다.
4. Web 자동 로그인 → `학습지` → `학습하기` → 문제 화면까지 이동한다.
5. `채점 끝내기`를 누르고 `QR_READY` 복귀를 확인한다.

합격 기준:

- [ ] 실제 종이 한 장 출력
- [ ] 종이 QR 1회 인식
- [ ] 올바른 시험계정 자동 로그인
- [ ] 문제 화면 진입과 정상 로그아웃
- [ ] 최종 `QR_READY`
- [ ] 프린터 대기열 잔류 없음

시험 뒤 종이 카드는 통제된 장소에 보관한다. 분실하거나 폐기할 때는 앱에서 다시 QR을 재발급해 종이 카드의 토큰을 무효화한다.

## 실패했을 때

QR을 반복 촬영하거나 Android 뒤로가기를 누르지 않는다. 아래 네 가지를 메모해 보고한다.

1. 실패한 단계
2. 화면의 상태 코드 또는 오류 문구
3. 연속 운전 경과시간
4. 화면 꺼짐·재부팅·Wi-Fi 끊김 여부

`LOCKED`, `WEB_SESSION_FAILED` 또는 `WEB_SESSION_NOT_CLEAN`이면:

1. 관리자 PIN 입력
2. `Web 세션 안전 정리` → `정리 시작`
3. `현재 수업 안전 종료`
4. `선택한 반 수업 시작`
5. `QR_READY` 확인

## 결과 보고 양식

아래 형식으로 대화에 결과만 적는다. 계정값·PIN·QR은 적지 않는다.

```text
연속 운전: PASS/FAIL, ____분
종료 왕복: PASS/FAIL
프린터: PASS/FAIL/SKIP
오류 코드: 없음 또는 ________
최종 상태: QR_READY / 기타 ________
```

연속 운전과 종료 왕복이 PASS면 무인 대기 검증은 완료다. 프린터를 SKIP하면 핵심 운영 상태에는 영향이 없지만 실제 출력·종이 인식은 미검증으로 기록한다.
