# Web Gate 3 실제 교차 기록

개인정보·자격정보·화면 내용 없이 실행 시각, 비민감 진행값, 상태와 일반화된 종료 사유만 기록한다.

|---:|---|---|---:|---:|---:|---|---|
| 1 | 2026-07-22 14:14:05 | 0.2.0 / `5BE869…DCD1` | 100 | 82 | 1,357,305 ms (22.62분) | `FAILED / LOCKED / LOGIN_TIMEOUT` | G301 FAIL |
| 2 | 2026-07-22 14:49:33 | 0.2.1 / `06A226…0EA` | 100 | 74 | 1,548,210 ms (25.80분) | `FAILED / LOCKED / LOGIN_TIMEOUT` | G301 FAIL |
| 3 | 2026-07-22 16:04:05 | 0.2.2 / `133751…E95` | 100 | 100 | 2,137,404 ms (35.62분) | `PASSED / IDLE / 사유 없음` | G301 PASS |

시도 1은 A/B를 정확히 교대해 성공한 cycle만 82회 계상했다. 따라서 A와 B는 각각 41회 정상 로그인·표시명 일치·자동 로그아웃·session 정리를 완료했다. 83번째 cycle의 로그인에서 30초 제한을 넘자 로그인 자동 재시도 없이 즉시 중단했고, 남은 18회는 실행하지 않았다. 종료 2초 뒤에도 `LOCKED / LOGIN_TIMEOUT`, Gate 3 status `FAILED`가 유지됐다.

실패 원인이 서버 응답 지연, 일시 네트워크 지연 또는 연속 인증 빈도 중 무엇인지는 민감 로그를 수집하지 않아 확정하지 않는다. 실패 폐쇄와 다음 계정 차단은 정상 동작했지만, 정상 교차 100/100 기준은 충족하지 못했다.

재시험용 0.2.1은 로그인 30초 timeout과 실패 폐쇄 기준은 유지하고, 검증 완료 cycle 사이 휴지시간만 0.75초에서 5초로 늘렸다. 전체 clean build와 A 계측 18/18, 설치 뒤 `IDLE`을 통과했다.

시도 2는 5초 휴지시간으로 A/B를 정확히 교대해 성공한 cycle만 74회 계상했다. 따라서 A와 B는 각각 37회 정상 로그인·표시명 일치·자동 로그아웃·session 정리를 완료했다. 75번째 cycle의 로그인에서 30초 제한을 넘자 로그인 자동 재시도 없이 즉시 중단했고, 남은 26회는 실행하지 않았다. 종료 2초 뒤에도 `LOCKED / LOGIN_TIMEOUT`, Gate 3 status `FAILED`와 74/100이 유지됐다.

시도 1과 시도 2 모두 장시간 정상 교차 뒤 단일 로그인 결과 확인 timeout으로 끝났다. 5초 휴지만으로 실패를 제거하지 못했으므로 연속 인증 빈도가 단독 원인이라고 단정할 수 없다. 다음 후보는 로그인 페이지 준비 제한을 유지하면서 제출 후 결과 확인 제한만 별도로 늘리는 것이다. 실제 100/100을 다시 통과하기 전 G301은 FAIL로 유지한다.

재시험용 0.2.2는 단일계정 흐름과 로그인 페이지 준비 timeout 30초를 유지한다. Gate 3에서 로그인 제출이 정상 수행된 뒤 결과를 기다리는 timeout만 60초로 분리하며, 로그인 자동 재시도 0·표시명 완전일치·실패 폐쇄·5초 cycle 휴지는 유지한다.

시도 3은 0.2.2로 A/B를 정확히 교대해 100/100을 완료했다. A와 B는 각각 50회 정상 로그인·표시명 일치·자동 로그아웃·빈 로그인 화면·session 정리를 완료했다. 종료 직후와 2초 뒤 모두 `PASSED / IDLE`, 완료 100/100, 사유 없음과 동일 소요시간이 유지됐다. preferences 키는 `state`, Gate 3 status/completed/target/duration뿐이었고 자격정보 키는 없었다.

이 결과로 정상 교차 G301은 PASS다. 다만 시도 3만으로 시도 1·2의 timeout 원인이 30초 제한이었다고 확정하지는 않는다. G302~G308 실패·복구 시험이 끝나기 전 Web Gate 3 전체 판정은 아직 PASS가 아니다.

시도 3 완료 뒤 앱 프로세스 강제종료·재실행과 A 전체 재부팅·사용자 unlock·앱 재실행을 각각 수행했다. 두 경우 모두 `PASSED / 100/100 / IDLE / 사유 없음`, 동일 소요시간과 비민감 저장 키 5개만 유지돼 G308은 PASS다.

0.2.3 설치 뒤 background/foreground 합성 입력 6개 삭제 계측과 A의 실제 화면 off/on·unlock을 수행했다. 실제 시험에서는 첫 이름란의 비자격정보 임시 문자만 사용했고 화면 복귀 뒤 값이 사라진 것을 사용자가 확인해 G307은 PASS다.
