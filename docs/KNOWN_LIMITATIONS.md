# 알려진 제한과 잔여위험

- Gate 1 최종 FAIL: 기억하기 상태, 로그인 직후/홈 이동, 문제풀이 복귀와 프로필 진입에 필요한 의미 기반 상태·Action이 실제 SM-P610/매쓰홀릭 7.5.1 접근성 트리에 노출되지 않았다.
- 접근성 의미노드 의존: 현재 결과를 우회하려면 좌표·이미지·비공개 방식이 필요하지만 승인 범위와 실패폐쇄 원칙에 맞지 않아 사용하지 않는다. 공급사가 접근성 role·label·action을 제공한 새 버전에서만 재검증한다.
- 서버 UI 변경: versionName이 같아도 화면이 바뀔 수 있다. version 검사만으로 안전하지 않아 런타임 fingerprint가 필요하다.
- 학생 식별: 프로필에서 이름·아이디·학원명 후보를 찾았지만 resource ID나 의미 role이 없고 다른 계정으로 값 변화만 교차검증하지 않았다. 생산 selector로 확정할 수 없다.
- 로그아웃 검증: 일반 앱은 서버 세션 폐기를 직접 증명할 수 없다. 로그인 화면과 빈 입력란에 의한 운영상 UI 검증이다.
- K1 잠금: 접근성 overlay와 화면 고정의 기존 한계는 유지된다. Gate 5 alpha는 공장초기화한 A에서 Device Owner·전용 HOME·Lock Task로 홈·최근 앱·알림창·설정을 차단했지만, 이 결과를 일반 앱인 `poc`나 비-Device Owner 설치에 적용할 수 없다.
- restricted settings: Android 13 이상에서 sideload 접근성 서비스를 켜려면 사용자가 앱 정보에서 제한된 설정을 허용해야 할 수 있다. 자동 우회하지 않는다.
- 보고서 fingerprint: salt가 세션마다 바뀌므로 다른 Probe 프로세스의 digest는 직접 비교할 수 없다. 같은 캡처 세션 내 중복 판별용이다.
- bounds: 기기/창 크기 진단을 위해 저장하지만 selector로 사용하지 않는다.
- A/B 결과: SM-P610 전체 Gate 1과 SM-P613 기억하기 단축 재검증에서 같은 필수 실패가 재현됐다. B는 해당 실패만으로 Gate 1 PASS가 불가능해 인증 후 화면 반복 수집을 생략했다.
- 외부 알림: 사용자 결정에 따라 현재 요구에서 제거했다. 태블릿 UI/소리/로컬 audit 대안은 후속 Gate에서만 설계한다.
- 공급사 허용: 자동화 사용 가능하다는 사용자 확인을 근거로 진행하지만 법률 의견이나 계약 해석은 아니다. 문의 답변 원본 보관을 권장한다.
- 대화 노출: 시험계정 비밀번호가 대화에 입력되었으므로 POC 완료 후 변경하는 것이 안전하다. 저장소에는 포함하지 않는다.
- Web POC 식별자 잔류: 공식 웹은 저장 checkbox가 해제되어도 로그아웃 뒤 이전 아이디를 다시 채운다. `webpoc`은 모든 로그인 페이지 로드에서 두 입력값을 비우고 checkbox를 해제한 뒤 재검증하지만, 이 동작이 실패하면 다음 학생 입력을 허용하지 않는다.
- WebView 임시 저장: 인증 중 access/refresh token이 공식 학습 웹의 WebView 저장소에 존재할 수 있다. 정상 로그아웃 뒤 Cookie/WebStorage/form data를 삭제하며, 민감 상태에서 재시작하면 안전 로그아웃·정리를 우선한다.
- Gate 5 alpha: A 한 대에서 Device Owner 잠금, QR→Web→QR 왕복과 재부팅 복구를 통과했다. 다른 제조사·Android 버전, 실제 학생 파일럿과 장시간 무인 운전 결과는 없다.
- Gate 5 패키징: 별도 release signer의 RC APK와 자동 검증은 준비했지만 A에는 아직 두 debug APK가 설치되어 있다. release 키의 휴대 가능한 복구 확인, 두 번째 공장초기화, release 실기, 서명키 회전과 원격 업데이트 채널은 완료하지 않았다.
- USB 디버깅: 프로비저닝과 검증을 위해 A의 USB 디버깅과 PC 승인을 유지했다. 승인된 PC의 ADB는 앱 삭제·데이터 초기화 등 Lock Task보다 강하므로 생산 투입 전 개발자 옵션과 USB 디버깅을 끄고 별도 실기 재검증해야 한다.
- Device Owner 복구: 관리자 PIN 분실이나 Device Owner 제거는 공장초기화가 필요하며 앱 DB, Keystore 키, 자격정보와 기존 QR을 복구하지 않는다.
- 관리자 PIN 지연은 앱 private DB의 wall clock 기준이다. 잠금 중 설정 진입은 차단하지만, 관리자 잠금 해제나 승인된 ADB를 가진 주체의 시각 변경·앱 데이터 삭제까지 방어하지 않는다.
- QR 인쇄: Gate 5는 명시적 경고 뒤 현재 QR을 Android 인쇄 서비스에 전달하고 즉시 앱 화면에서 제거한다. `FLAG_SECURE`는 앱 밖의 인쇄 대기열, 프린터 메모리와 PDF 저장 대상을 보호하지 않는다. A에서 ISO A4 1페이지 미리보기는 통과했지만 실제 프린터 출력은 하지 않았다. 통제된 로컬 프린터만 사용하고 잔류 작업과 분실 카드는 즉시 폐기해야 한다.
- 학생 비활성화: 기존 QR hash를 새 무작위 hash로 교체하고 활성 조회에서 제외하는 논리적 비활성화다. 앱 private DB의 암호화된 자격정보 레코드를 보안 삭제하지는 않는다.
- 카메라: `0.5.0-alpha05`는 전면을 기본으로 사용하고 화면 버튼으로 후면 전환 및 다시 전면 복귀가 가능하다. A에서 두 렌즈 바인딩과 전면 실제 QR 왕복은 통과했지만 다른 모델의 전면 카메라 화질·초점·조명 성능은 측정하지 않았다.
