# Web Gate 3 시험계획

## 범위

Gate 3는 실제 학생이 아닌 서로 다른 표시명의 학생용 시험계정 A·B 두 개만 사용한다. QR, 실제 학생 DB, 관리자 PIN, 외부 알림과 기기 잠금은 아직 포함하지 않는다.

자격정보는 A 태블릿 화면에서 한 번만 입력한다. 두 계정의 표시명·아이디·비밀번호는 100회 실행 동안 프로세스 메모리에만 유지하며 파일, preferences, 로그, fixture와 ADB 명령으로 전달하지 않는다. 완료·실패·중단·프로세스 종료 시 메모리 배열을 덮어쓰고 참조를 폐기한다. JVM·WebView 내부의 일시적 문자열 복사본까지 즉시 제거됐음을 증명할 수는 없다.

## 자동 교차 흐름

1. 공개 로그인 페이지 preflight와 빈 입력·저장 checkbox 해제 확인
2. 계정 A 로그인 제출 1회
3. 실제 표시명과 A의 예상 표시명 완전 일치 확인
4. 750 ms ACTIVE 확인 뒤 자동 로그아웃
5. 로그인 화면·빈 입력·checkbox 해제와 WebView session 삭제 재검증
6. 검증 성공 뒤에만 완료 회수 증가
7. 다음 인증 전 5초 휴지
8. 계정 B로 같은 흐름 실행
9. `A→B→A→B`를 100회까지 반복

로그인 자동 재시도는 0회다. 기존 안전계약에 따라 로그아웃은 최대 1회만 재시도한다. terminal 실패가 발생하면 즉시 `LOCKED` 또는 `MAINTENANCE_REQUIRED`로 중단하고 남은 회차를 실행하지 않는다.

## 저장 가능한 증거

SharedPreferences에는 다음 비민감 값만 저장한다.

- 앱 상태와 일반화된 종료 사유
- Gate 3 상태: `RUNNING`, `PASSED`, `FAILED`, `ABORTED`
- 완료 회수, 목표 회수, 총 소요시간

계정 슬롯은 화면 진행 표시에서 `A` 또는 `B`로만 나타나며 preferences에 저장하지 않는다. 프로세스가 끝난 `RUNNING` 시험은 다음 실행에서 `ABORTED`로 바뀌고 자동 재개되지 않는다.

## 필수 시험

| ID | 시험 | 기대 결과 |
|---|---|---|
| G301 | 정상 A↔B 교차 | 100/100, A·B 정확한 교대 |
| G302 | A에 B expectedName 매핑 | 학습 허용 전 로그아웃, `STUDENT_MISMATCH`, 즉시 중단 |
| G303 | 한 계정 비밀번호 변경 뒤 과거 값 | 제출 1회, 자동 재시도 0, 즉시 중단 |
| G304 | 이전 계정 세션 잔류/ACTIVE 종료 | 복구 로그아웃 전 다음 계정 로그인 0 |
| G305 | 로그인·로그아웃 중 프로세스 종료 | 자동 재개 0, `ABORTED`, 안전상태 재검증 |
| G306 | 네트워크 단절·서버지연 | timeout 또는 network lock, 다음 계정 로그인 0 |
| G307 | 화면 off/on·background/foreground | 자격정보 입력 화면 재노출 0 |
| G308 | 완료 직후 앱·기기 재시작 | 자격정보 0, 안전 `IDLE` |

## PASS 기준

- 정상 전환 100/100
- 다른 학생 계정 허용 0
- 실패를 성공으로 오판 0
- 로그아웃 검증 전 다음 로그인 0
- 로그인 자동 재시도 0
- 자격정보 파일·preferences·로그·ADB 전달 0
- 중단·프로세스 종료 뒤 자동 재개 0

실제 시도 1은 82/100, 시도 2는 74/100 뒤 각각 `LOGIN_TIMEOUT`으로 실패 폐쇄됐다. 시도 2에서 cycle 사이 휴지를 0.75초에서 5초로 늘렸지만 같은 종료 사유가 재현됐다. Gate 3의 제출 후 결과 확인만 60초로 분리한 0.2.2 시도 3은 100/100, A·B 각 50회를 완료해 G301을 통과했다. 이후 G306·G307·G308과 G304·G305 자동 중단 계약을 통과했다. 네트워크 경로가 변경된 0.3.2도 실제 정상 100/100(A·B 각 50회) 회귀를 통과했다. G302는 실제 표시명 교차 매핑에서 `STUDENT_MISMATCH`, 완료 0, B 로그인 0으로 통과했다. G303은 0.3.3 수정 뒤 `LOGIN_NOT_VERIFIED`, 완료 0, B 로그인 0으로 통과했고, 실계정 session 기반 G304·G305의 `LOGIN_SUBMIT`·`ACTIVE`·`LOGOUT_SUBMIT` 종료도 모두 완료 회수 불변과 `ABORTED / IDLE`을 통과했다. 최종 0.3.3 자체의 정상 100회 회귀 전에는 Gate 3 전체 PASS로 판정하지 않는다. 세부 결과는 `WEB_GATE3_RUN_LOG.md`에, runner 구현과 비자격정보 검증은 `WEB_GATE3_VERIFICATION.md`에 기록한다.
