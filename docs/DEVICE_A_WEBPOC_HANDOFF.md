# 기기 A Web POC 즉시 이관 절차

## 준비 상태

기기 A의 기준 모델은 `SM-P610`이다. 2026-07-22 A에서 전체 이관 절차와 로그아웃 보강 후 Web POC 계측시험 15/15, 본체 재설치와 최종 `IDLE`을 확인했다. 앞서 B(`SM-P613`)를 명시적으로 대상으로 삼은 일반화 스크립트 회귀시험도 당시 12/12와 최종 `IDLE`을 통과했고, A 전용 명령이 B를 A로 오인하지 않고 설치 전에 중단하는 것도 확인했다.

## A에서 할 일

1. A의 잠금을 해제하고 USB 디버깅을 켠다.
2. 이 PC의 USB 디버깅을 허용한다. USB 모드는 파일 전송이면 충분하다.
3. A의 Wi-Fi가 연결됐는지 확인한다.
4. 저장소 루트에서 다음 한 명령을 실행한다.

```powershell
.\scripts\prepare-a-webpoc.ps1
```

이 명령은 순서대로 다음을 수행한다.

1. 연결된 승인 기기 중 모델이 정확히 `SM-P610`인 A 하나를 선택한다.
2. 전체 clean 빌드와 단위시험·lint를 실행한다.
3. 일련번호·계정·화면·logcat을 제외한 비민감 기준정보를 수집한다.
4. `ANDROID_SERIAL`을 A로 한정하고 Web POC 계측시험 15개를 실행한다.
5. 시험 APK를 제거한 뒤 Web POC 본체를 A에 다시 설치하고 실행한다.
6. 앱의 비민감 상태값이 45초 안에 실제 `IDLE`에 도달했는지 확인한다.

A와 B가 동시에 연결되어 있어도 `SM-P610` 하나만 선택한다. A가 없거나 둘 이상의 `SM-P610`이 연결됐거나 USB 인증이 끝나지 않았으면 빌드·설치 전에 중단한다. 잘못된 모델, 빌드·시험 실패, 설치 실패, `LOCKED` 또는 `MAINTENANCE_REQUIRED`도 성공으로 보고하지 않는다.

## 선택 옵션

이미 같은 소스의 전체 빌드가 끝난 경우에만 빌드를 생략할 수 있다.

```powershell
.\scripts\prepare-a-webpoc.ps1 -SkipBuild
```

계측시험을 생략하고 설치만 하는 옵션도 있지만 최종 A 검증에는 사용하지 않는다.

```powershell
.\scripts\prepare-a-webpoc.ps1 -SkipDeviceTests
```

## 설치 뒤 실제 시험

스크립트가 성공한 뒤에만 태블릿 화면에서 예상 표시명·시험 아이디·시험 비밀번호를 직접 입력한다. PC, ADB, 소스, 파일 또는 로그로 자격정보를 전달하지 않는다. 실제 로그인→표시명 일치→문제화면→`채점 끝내기`→자동 로그아웃을 먼저 1회 확인하고, 이후 정상 20회와 실패주입 계획을 진행한다.

Web POC에는 접근성 권한, overlay, Device Owner 또는 Android 매쓰홀릭 앱 설치가 필요하지 않다. 기존 Gate 1 Probe 접근성 서비스가 남아 있다면 Web POC 시험 전 수동으로 꺼도 된다.
